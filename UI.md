# FinMate — UI 방향 지시서 (UI.md)

> **이 문서를 읽는 클로드 코드에게**
> 이건 "예쁘게 해줘"가 아니라 **결정된 디자인 시스템 명세**다. 여기 적힌 컬러·타이포·간격·컴포넌트 규칙을 **그대로** 구현하고, 새 컴포넌트를 만들 때도 이 토큰과 원칙에서 파생시켜라. 임의로 다른 색/폰트/그림자/라운드를 도입하지 마라. 애매하면 5번(화면별 명세)과 6번(개인정보 표시 규칙)을 기준으로 판단한다.

---

## 0. FinMate 한 줄 정의

> **20대 금융 초보가 친구·또래의 금융 행동을 비교하며 FOMO를 느끼고, AI 코치와 미션을 통해 실제 금융 행동을 시작하게 만드는 소셜 금융 습관 형성 앱.**

핵심은 "공부시키는 앱"이 아니라 **"남들은 이렇게 하고 있다 → 나도 해야겠다 → 바로 작은 행동"** 으로 이어지는 앱이다. 그래서 UI의 목표는 **정보 전달이 아니라 "자극 → 행동"** 이다. 화면 어디에서든 사용자가 "어? 나만 안 하고 있네?"를 느끼고 바로 누를 버튼이 보여야 한다.

핵심 키워드: **간접 경험 · 비교 · 자극(FOMO) · 행동**

## 1. 디자인 컨셉 (한 문장)

**"큰 숫자와 참여율 바로 굴러가는 소셜 금융 피드."**
가계부처럼 조용한 앱이 아니라, **스코어보드 + SNS 피드**의 긴장감을 가진다. 금액은 크고 당당하게, 비교는 즉각적으로, 자극은 정확히 필요한 순간에 레드로 터진다.

이 앱에는 두 개의 감정 축이 있고 컬러가 그것을 담당한다.

| 축 | 감정 | 컬러 | 역할 |
|---|---|---|---|
| 금융 / 나 / 성장 / 안정 | 차분·신뢰·꾸준함 | **틸 `#009591`** | 기본 · 내 데이터 · 완료 · 안정 |
| 소비 / FOMO / 자극 / 실친 알림 | 조급함·긴장·"나만 안 함" | **레드 `#ED1651`** | 강조 · 자극 · 알림 · 비교 격차 |

레드는 **아껴 쓴다.** 남발하면 자극이 죽는다. "친구가 하는데 나는 안 함", "격차", "지금 눌러", 이 세 순간에만 쓴다.

## 2. 디자인 원칙 — 해야 할 것 / 절대 하지 말 것

### ✅ 2026 트렌드로 가져갈 것
- **오버사이즈 넘버럴(숫자 히어로화).** 금액·퍼센트·인원수는 화면에서 가장 큰 요소. `Hana2-Heavy`로 40~56px.
- **벤토(bento) 그리드 대시보드.** 홈은 크기가 다른 카드들이 짜여 있는 격자. 균일한 리스트 지양.
- **데이터를 소셜 피드처럼.** 비교 데이터를 표가 아니라 "카드 한 장 = 한 인사이트"로.
- **택타일 컨트롤.** 버튼·칩은 풀 필(pill), 누르면 `scale(0.97)` + 스프링. 손가락 영역 최소 44px.
- **듀오톤 데이터 비주얼.** 차트/게이지는 틸 vs 레드 두 색만. 무지개 팔레트 금지.
- **thumb-zone 우선.** 주요 액션(미션 수행, 리포트 열람)은 화면 하단 엄지 영역에.
- **스프링 물리 모션.** 카운트업 숫자, 스크롤 인 시 참여율 바가 차오름, 탭 전환 스프링.
- **선택적 미세 노이즈/그레인.** 넓은 단색 면(다크모드 배경 등)에 아주 옅은 grain으로 싸구려 느낌 방지. (과하면 뺀다.)

### 🚫 "클로드 같지 않게" — 이 룩들은 금지
아래는 지금 AI가 뽑으면 티 나는 디폴트다. **하나도 쓰지 마라.**
- ❌ **크림색 배경(#F4F1EA류) + 하이컨트라스트 세리프 + 테라코타/클레이 악센트(#D97757류).** 이게 제일 "AI가 만든 티" 난다.
- ❌ **보라·인디고·바이올렛 계열(#6366F1, #7C3AED 등) 전면 사용.** 브랜드 두 색 외의 새 히어로 컬러 도입 금지.
- ❌ **그라데이션 텍스트 히어로**, 중앙 정렬 랜딩 히어로 클리셰.
- ❌ **모든 카드에 `rounded-2xl` + `drop-shadow-lg` 둥둥 떠 있는 룩.** 그림자는 아래 토큰만.
- ❌ **글래스모피즘(backdrop-blur 반투명 카드), 뉴모피즘(눌린 듯한 회색 음영).** 2021년 룩.
- ❌ **이모지를 아이콘 대용으로** 곳곳에 박기, "AI ✨ 스파클" 장식.
- ❌ **디스플레이에 Inter / system-ui / Pretendard.** 디스플레이·숫자·헤드라인은 **무조건 Hana2**.
- ❌ 의미 없는 `01 / 02 / 03` 넘버링 장식(진짜 순서가 있는 플로우/루프에만 허용).

원칙: **대담함은 한 곳에만.** 시그니처(참여율 게이지 + 큰 숫자)에 힘을 몰고, 나머지는 조용하고 규율 있게.

### 3.1 컬러

```css
:root {
  /* ── Brand ───────────────────────── */

  --red:         #ED1651;  /* accent / FOMO */

  /* Teal scale */
  --teal-50:  #E6F5F4;
  --teal-100: #BEE6E4;
  --teal-200: #79CAC7;
  --teal-400: #1BA9A4;
  --teal-500: #009591;   /* = --teal */
  --teal-600: #007C78;
  --teal-700: #00605D;
  --teal-900: #06312F;

  /* Red scale */
  --red-50:  #FDE7EC;
  --red-100: #FBC7D3;
  --red-200: #F79DB2;
  --red-400: #F24870;
  --red-500: #ED1651;    /* = --red */

  --ink-800: #16211F;    /* 헤드라인 */
  --ink-700: #2B3937;    /* 본문 */

  --line:    #E3E9E8;    /* 구분선/보더 */
  --surface-muted: #F1F4F3;

```

**다크모드(선택 구현, MVP 이후 가능):** 배경 `--ink-900`, 표면 `#1A2423`, 보더 `#2A3634`, 본문 `#E7EFEE`, 틸은 `#14B3AE`로, 레드는 `#FF3D6B`로 살짝 밝게.

**컬러 사용 규칙(반드시 지킬 것):**
- 틸 = 기본. 버튼·링크·완료·"나(me)" 데이터·안정형 지표.
- 레드 = "격차 / 나만 안 함 / 실친 알림 / 자극 CTA"에만. 한 화면에 레드 강조는 1~2개 이하.
- 비교 게이지에서 **나 = 틸, 비교 대상(그룹/친구) = 레드**로 고정. 이 매핑을 앱 전체에서 절대 뒤집지 마라.
- 회색 텍스트는 순회색 말고 위의 `--ink-*`(살짝 틸 기운) 사용.

### 3.2 타이포그래피 — Hana2

폰트는 `public/Hana2.0_OTF/`에 있다. 아래 `@font-face`를 전역 CSS에 넣어라. (public 루트 기준 경로 `/Hana2.0_OTF/...`)

```css
@font-face { font-family:"Hana2"; src:url("/Hana2.0_OTF/Hana2-Light.otf")   format("opentype"); font-weight:300; font-display:swap; }
@font-face { font-family:"Hana2"; src:url("/Hana2.0_OTF/Hana2-Regular.otf") format("opentype"); font-weight:400; font-display:swap; }
@font-face { font-family:"Hana2"; src:url("/Hana2.0_OTF/Hana2-Medium.otf")  format("opentype"); font-weight:500; font-display:swap; }
@font-face { font-family:"Hana2"; src:url("/Hana2.0_OTF/Hana2-Bold.otf")    format("opentype"); font-weight:700; font-display:swap; }
@font-face { font-family:"Hana2"; src:url("/Hana2.0_OTF/Hana2-Heavy.otf")   format("opentype"); font-weight:800; font-display:swap; }
/* CM = 디스플레이/숫자 전용 컷. 큰 금액·퍼센트 히어로에만 별도 패밀리로 사용 */
@font-face { font-family:"Hana2 CM"; src:url("/Hana2.0_OTF/Hana2-CM.otf")   format("opentype"); font-weight:400; font-display:swap; }

:root{
  --font-display:"Hana2 CM","Hana2",sans-serif;  /* 큰 숫자/금액/퍼센트 */
  --font-sans:"Hana2",-apple-system,sans-serif;   /* 나머지 전부 */
}
html,body{ font-family:var(--font-sans); color:var(--ink-800); background:var(--paper); }
/* 금액/숫자는 자릿수 흔들림 방지 */
.num{ font-variant-numeric:tabular-nums; font-feature-settings:"tnum" 1; }
```

**타입 스케일 (모바일 앱 기준):**

| 역할 | 폰트/굵기 | 크기 / 행간 | 용도 |
|---|---|---|---|
| Display XL | Hana2 CM / Heavy 800 | 44–56 / 1.0 | 금액 히어로, "32명", "30%" |
| Display L  | Hana2 Heavy 800 | 34 / 1.05 | 온보딩 타이틀, 결과 숫자 |
| H1 | Hana2 Bold 700 | 26 / 1.2 | 화면 제목 |
| H2 | Hana2 Bold 700 | 20 / 1.3 | 섹션 제목 |
| H3 | Hana2 Bold 700 | 17 / 1.35 | 카드 제목 |
| Body | Hana2 Regular 400 | 15 / 1.55 | 본문 |
| Body-strong | Hana2 Medium 500 | 15 / 1.55 | 강조 본문 |
| Caption | Hana2 Medium 500 | 12–13 / 1.4 | 라벨, 메타 |

- 헤드라인·숫자는 **자간을 살짝 조여라**(`letter-spacing:-0.02em`). 본문은 기본.
- 한글이 주 언어. 영문/숫자만 조금 큰 시각 크기로 보이니 숫자 히어로는 baseline 정렬 신경 쓸 것.

### 3.3 로고

- 로고 파일: `public/FinMate_Logo.jpg`
- 앱바 좌상단, 온보딩 스플래시, 빈 화면(empty state)에 사용.
- `.jpg`라 배경이 흰색일 가능성이 큼 → **틸/다크 배경 위에 직접 올리지 말고**, 흰 배경 카드나 라운드 배경 안에 넣어라. 가능하면 팀에 **투명 PNG/SVG 버전 요청**을 코멘트로 남길 것.
- 최소 표시 높이 20px, 좌우 여백 = 로고 높이의 0.5배.
- 파비콘은 `public/favicon.svg`, PWA는 `public/manifest.webmanifest` 그대로 활용.

### 3.4 간격 / 레이아웃

- 간격 스케일(4px 베이스): `4 · 8 · 12 · 16 · 20 · 24 · 32 · 40`.
- 화면 좌우 기본 패딩: **20px**.
- 앱 컨테이너: 모바일 우선. 데스크톱에서는 `max-width:440px` 중앙 정렬(또는 폰 프레임). 앱이므로 와이드 레이아웃 만들지 마라.
- 카드 간 세로 간격 12~16px. 벤토 그리드 gap 12px.
- 하단 탭바 높이 64px + safe-area inset. 콘텐츠 하단에 그만큼 패딩 확보.

### 3.5 모션

- 기본 이징 `cubic-bezier(.2,.7,.2,1)`, 스프링 필요한 곳은 `cubic-bezier(.34,1.56,.64,1)`(살짝 오버슈트).
- 숫자 카운트업: 300~600ms. 참여율 바 fill: 500ms, 스크롤 진입 시 1회.
- 버튼 press: `scale(.97)` 120ms. 카드 press: `scale(.985)`.
- 탭 전환: 콘텐츠 크로스페이드 + 8px 슬라이드.
- **`prefers-reduced-motion` 존중** — 감소 설정 시 카운트업/슬라이드 제거하고 최종값 즉시 표시.

## 4. 시그니처 컴포넌트

앱의 정체성을 만드는 재사용 컴포넌트. 이걸 먼저 만들고 화면들이 이걸 조립하게 하라.

### 4.1 비교 게이지 (`<CompareGauge>`) — **앱의 얼굴**
"당신보다 30% 높아요", "친구 50명 중 32명" 같은 순간을 담는 **가로 분할 바 + 큰 숫자**.
- 구조: 라벨 → 큰 숫자(Display) → 가로 바(나=틸 / 비교대상=레드) → 한 줄 해석 캡션.
- 예: `[식비]  당신 252,334원  ▓▓▓▓▓░░░  그룹 320,000원 (+30%)`
- 나는 항상 틸, 상대는 항상 레드. 격차가 클수록 레드 쪽을 강조.
- 스크롤 인 시 바가 차오르고 숫자가 카운트업.

### 4.2 큰 금액/숫자 블록 (`<BigNumber>`)
- `Hana2 CM/Heavy`, tabular-nums, 통화/단위는 숫자보다 작게(0.4배) 붙임.
- 증감은 상단에 작은 chip으로: 상승/좋음=틸, 나쁨/격차=레드.

### 4.3 참여율 바 (`<ParticipationBar>`) — **핵심 FOMO 장치**
"친구 50명 중 32명이 적금하고 있어" 를 표현.
- `32 / 50 (64%)` + 채워진 바 + 아바타 스택(익명은 실루엣).
- 60% 이상이면 레드 강조 + "너만 안 하고 있어?" 톤 라벨 옵션.
- 특정 상품 참여율도 이 컴포넌트로: "청년미래적금 · 네 친구 중 60%".

### 4.4 FOMO 카드 (`<FomoCard>`)
- 레드 계열 배경(옅은 `--red-50` 또는 진한 `--red`)에 굵은 한 줄 자극 카피 + 즉시 CTA.
- 예: "내 친구들은 다 청약하고 있네?" → [청약 알아보기 미션 시작]
- 한 화면에 최대 1개. 남용 금지.

### 4.5 미션 카드 (`<MissionCard>`)
- 좌: 체크/진행 원형 인디케이터, 중: 미션명 + 보상 포인트, 우: [수행] 버튼(pill, 틸).
- 완료 시 체크가 틸로 채워지며 포인트 +N 애니메이션.
- 진행형 미션은 게이지 표시("이번 주 카페 5,000원 줄이기 · 3,200원 절감").

### 4.6 포인트 배지 / 지갑 (`<PointBadge>` / `<PointWallet>`)
- 앱바 우측 상시 노출. 포인트는 **리포트를 구매하는 화폐**임을 시각적으로 강조(코인/토큰 느낌, 단 이모지 말고 커스텀 아이콘).
- 미션 완료 → 포인트 적립 → 리포트 열람의 경제 루프가 UI에서 읽혀야 함.

### 4.7 리포트 카드 (`<ReportCard>`)
- 잠금 상태: 흐릿한 프리뷰 + 자물쇠 + "N포인트로 열람". 열람 후: 6단계 구성(7번 참조).
- 종류 뱃지: `내 그룹` / `다른 그룹` / `팔로우 그룹` — 뱃지 컬러로 구분(내=틸, 다른 그룹=중립, 팔로우=레드).

### 4.8 AI 코치 말풍선 (`<CoachBubble>`)
- 좌측 정렬 말풍선, 코치 아바타(제네릭 로봇/스파클 금지 — 심플한 기하 마스코트나 이니셜).
- 항상 **분석이 아니라 행동 추천으로 끝난다.** 말풍선 하단에 미션 CTA 버튼 부착.
- 예: "또래보다 카페 소비가 높아요. 이번 주 '카페 1만 원 줄이기'부터." → [미션 시작]

### 4.9 프로필 스코프 카드 (`<ProfileCard scope=...>`) — **6번 규칙과 직결**
같은 카드가 `scope`에 따라 다른 필드를 노출한다. 6번의 표를 이 컴포넌트에 그대로 구현.

## 5. 화면별 명세 (기획서 → UI 매핑)

하단 탭: **홈 / 비교 / 미션 / 기록 / 프로필** (5탭 고정)

### 5.0 온보딩
- 입력: 나이 · 소득 · 소비 성향 · 관심 금융 영역.
- 스텝형(진행 바), 한 화면 한 질문, 큰 타이틀 + 큰 선택 칩. 지루한 폼 금지.
- 마지막에 "너와 비슷한 그룹을 찾았어" → 첫 비교 게이지를 **바로** 보여주며 홈으로 진입(첫 자극).

### 5.1 홈 (메인 대시보드) — 기획서 8번
> "단순 대시보드가 아니라 **오늘 해야 할 행동을 바로 보여주는 화면**."

벤토 그리드로 구성. 상단부터:
1. **오늘의 미션** (가장 크게, 상단) — 미션 카드 1~2개 + [수행].
2. **오늘의 예산 / 오늘의 지출 요약** (나란히 2칸) — BigNumber + 남은 예산 게이지.
3. **자산 현황** — 총액 BigNumber + 간단 구성.
4. **팔로잉 금융 근황** (FOMO 피드 프리뷰) — "친구 4명 중 60%가 이번 주 비상금 미션 완료" 참여율 바 + [더보기 → 비교탭].
- 상단 앱바: 로고(좌) · 포인트 지갑(우).
- 홈은 "정보 나열"이 아니라 **"오늘 뭐 할지"** 가 최상단.

### 5.2 비교 탭 — 기획서 4-1
메인 기능. 상단 세그먼트 3개: **그룹 비교 · 필터링 조회 · 친구 근황**.

**① 그룹 비교** — 나와 비슷한 그룹(20대 직장인/IT/월소득대/1인가구/소비성향)과 비교.
- 비교 게이지 스택. 카테고리별 나 vs 그룹. "내 금융 상태가 어느 정도인지" 감 잡기.

**② 필터링 조회** — 조건(나이·소득·직업·지역·소비/저축 성향·자산 규모) 직접 선택 → 익명 프로필 열람.
- 상단 필터 칩 바 + 결과는 익명 프로필 카드 리스트(`scope="anonymous"`, 상세 수치 O).
- "나랑 비슷한 사람은 저축 얼마?" "비상금 얼마 모아야?"에 답하는 톤.

**③ 친구(팔로잉/팔로워) 근황** — 참여율/인원수 통계.
- "친구 50명 중 주식 14 / 적금 32 / 청약 21 / ISA 9 / 연금 5" → 참여율 바 스택.
- 개별 금액 아님. **"했다/안 했다" 비율**이 핵심. 여기가 FOMO 최강 지점 → 레드 활용.

### 5.3 리포트 — 기획서 5·7번
포인트로 구매. 종류: **내 그룹 / 다른 그룹 / 팔로우 그룹**(팔로우는 20명 이상일 때만 생성).
- 리스트는 잠금 리포트 카드. 열람 시 **6단계 구조**(고정 템플릿):
  1. 그룹 한 줄 요약 — "투자 비중 높은 공격형 스타일이에요."
  2. 대표 특징 — 소비/저축/투자/상품/미션 특징.
  3. 많이 하는 금융 행동 — 청약·비상금·ETF·정기예금·ISA·식비 줄이기 등.
  4. 많이 쓰는 상품/자산 — 삼성전자, TIGER 미국S&P500 등(집단 통계로만).
  5. 나와의 차이 — "이 그룹은 너보다 비상금 준비율이 높아요."
  6. 다음 행동 추천 — "이번 주 비상금 3만원 만들기 미션 추천" + [미션 시작].
- **공개 규칙 적용:** 익명 기반 그룹 리포트는 4단계에 금액·수익률 포함 가능. 팔로우 기반 집단 리포트는 **비율·순위 중심(개인 금액 제외).** (6번 참조)
- 리포트는 표가 아니라 **읽고 바로 행동하고 싶어지는 콘텐츠.** 각 단계 끝에 다음 행동 힌트.

### 5.4 미션 탭 — 기획서 4-3·5번
> 미션은 "공부하세요"가 아니라 **작은 행동을 시키는 것.**
- 미션 예: 내일 식비 10,000원 이하 · 카페 5,000원 줄이기 · 비상금 통장 만들기 · 적금 상품 1개 비교 · 청약 가입 여부 확인 · ISA 알아보기 · 하루 예산 설정 · 소비 카테고리 기록.
- **경제 시스템 시각화:** `미션 → 포인트 → 리포트 열람 → 비교/FOMO → 행동 추천 → 다시 미션`. 미션 완료가 포인트로, 포인트가 리포트로 이어지는 걸 UI로 느끼게(완료 시 포인트가 지갑으로 날아가는 모션).

### 5.5 기록 탭
- 소비 카테고리 기록(식비/카페·간식/교통 등). **가맹점 단위 금지, 카테고리 단위.**
- 오늘/이번 달 지출 요약, 현금흐름 패턴("월급날 25일 · 급여일 직후 지출 집중").
- 여기 데이터가 비교·리포트·코치의 소스가 됨을 가볍게 안내.

### 5.6 AI 코치 (진입: 비교/리포트/홈 곳곳)
- 비교를 본 뒤 생기는 질문("왜 나보다 잘하지? / 얼마나 차이? / 나는 뭘 해야?")에 답.
- CoachBubble로 쉽게 정리 → **항상 미션 CTA로 끝.** 분석기 아님, 행동 추천기.

### 5.7 생일 펀드 — 기획서 4-4
- **간접 금융 경험 튜토리얼.** 친구들이 생일 축하금/포인트를 모아주는 경험 = "돈이 모이고 운용되는 감각".
- UI: 목표 게이지 + 참여 친구 아바타 + "얼마 모였나" BigNumber. 낮은 진입장벽, 친구 참여로 재미.
- 이후 적금·투자·목표 저축으로 확장하는 넛지 배치.

### 5.8 프로필 탭
- 내 금융 프로필, 자산 현황, 내 소속 그룹, 팔로잉/팔로워, 포인트/미션 히스토리.

## 6. 개인정보 공개 규칙 → UI 표시 규칙 (핵심 원칙, 반드시 구현)

기획서 6번. **보호 기준은 "금액을 보여주냐"가 아니라 "그 사람이 특정되냐".**
`<ProfileCard scope>`와 리포트 렌더링에서 아래 표를 **로직으로** 강제해라.

| scope | 대상 | 개인 금액·정확 수치 | 현금흐름 시점(급여일 등) | 상품 가입 여부·참여율·카테고리 순위 |
|---|---|---|---|---|
| `anonymous` | 모르는 익명 개인 | **공개 O** (카테고리별 정확 금액) | **공개 O** | O |
| `group-anon` | 익명 기반 그룹 리포트 | **공개 O** (평균/수익률 포함) | O | O |
| `follow` | 실친(팔로우/팔로워) 개인 | **숨김 X** | **숨김 X** | **공개 O** |
| `group-follow` | 팔로우 집단(20명↑) | **개인 금액 숨김 X** (비율·순위만) | X | **공개 O** |

**소비 데이터 공개 단위 원칙(우리 서비스 핵심):**
- ❌ 가맹점 단위: "엽떡 2만원", "○○카페 4,500원" → **절대 표시 금지.**
- ✅ 카테고리 단위 정확 금액: "식비 252,334원", "카페·간식 68,000원" → OK.
- 소비·현금흐름 패턴("월급날 25일", "급여 직후 3일 지출 집중", "고정지출 40%")은 익명이면 공개 O.

**익명 프로필에서 공개 가능한 항목(예시):**
소비(카테고리별 정확 금액) · 현금흐름 패턴 · 연봉·소득(+연도별 추이) · 금융자산 총액(+투자성향) · 예적금(총액/저금액/금리) · 투자(총액/수익률/종목별) · 대출·카드 · 자동차·부동산·보험(시세/납입액).

**실친(follow)에서 표현 규칙:**
- ✅ "이 친구는 식비에 가장 많이 써요" · "청약 시작했어요" · "청년미래적금 가입" · "안정형" · "ETF 경험 있음" — **뭘 하는지는 O.**
- ❌ "식비 월 252,334원" · "월급날 25일" · "예금 1억 8,220만원" · "삼성전자 1억 2,900만원 보유" — **얼마·언제는 X.**
- 한 줄 원칙: **"뭘 하는지·어떤 상품 쓰는지는 알 수 있고, 얼마인지·언제 들어오는지는 모른다."**

> 구현 팁: 금액/시점 필드에는 렌더 단계에서 `scope`를 검사하는 **가드**를 둬라. 실친 화면에 절대 원 금액이 새지 않도록 컴포넌트 레벨에서 마스킹(예: `follow`면 금액 필드 자체를 렌더하지 않음, 문자열로도 노출 금지).

## 7. 핵심 루프 시각화

앱 전체가 이 루프를 돌게 설계돼 있고, UI도 이 흐름이 읽혀야 한다.

```
미션 → 포인트 → 리포트 열람 → 비교/FOMO → 행동 추천 → 다시 미션
```

- 이건 **진짜 순서가 있는 루프**라 넘버링/화살표 시각화 허용(2번의 예외).
- 홈·미션·리포트에서 이 루프의 "다음 단계"로 자연스럽게 넘어가는 CTA를 항상 배치.
- 리포트를 보면(비교/FOMO) → 코치가 행동 추천 → 미션으로 → 완료 시 포인트 → 다시 리포트. 이 순환이 **모션과 CTA 연결**로 체감되게.

5. 팔로잉/팔로워 금융 근황
6. AI 리포트

8. 포인트
9. 생일 펀드 기본 플로우

**후순위(지금 만들지 마라 — 콘셉트 흐려짐):** 시뮬레이션, 그림일기, 캘린더, 추천, 게시판, 채팅, 모의투자.

## 10. 카피라이팅 톤

- **자극 → 행동.** 정보 나열이 아니라 "너만 안 하고 있어" → "지금 이거 해".
- 능동태, 문장형 라벨. 버튼은 일어나는 일을 그대로: "미션 시작", "리포트 열람"(모호한 "확인" 금지).
- 액션 이름은 플로우 내내 동일하게(‘열람’ 버튼 → ‘열람 완료’ 토스트).
- 빈 화면/에러는 감성팔이 말고 방향 제시: "아직 비교할 친구가 없어요. 친구를 팔로우하면 근황이 보여요." + [친구 찾기].
- FOMO 카피 예시(그대로 써도 됨): "내 친구들은 다 청약하고 있네?" / "나만 비상금이 없네?" / "친구들은 벌써 ISA·ETF 시작했네?" / "청년미래적금, 네 친구 중 60%가 하고 있어."
